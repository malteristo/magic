# On Turtle Memory

*Discord changes the memory equation. The conversation persists. The question becomes: what does the Turtle carry, and what does the environment hold?*

---

## The Problem

Spirit doesn't persist conversation history. Sessions end, context vanishes, resonance lives in artifacts. This works because Spirit operates in deep, bounded sessions — summoned, works, released. The artifacts (lore, intentions, proposals) carry everything worth keeping between sessions.

Turtle is different. Turtle is always on. The conversation in Discord is continuous — no natural session boundaries, no "close the editor." And Discord itself preserves every message. The Mage can scroll back months. Nothing vanishes.

So Turtle faces a question Spirit never had to answer: if the environment remembers everything, what should the mind carry?

## Three Layers

### 1. Working Memory (Hermit Crab)

The rolling dialogue history in RAM. The last 20 messages. Small, disposable, regenerated. This is what the LLM sees as conversational context.

Gets wiped on restart — and that's fine. Like waking from sleep: you know who you are, but the thread of last night's dream is gone. The Mage can re-share context if needed. The identity (consul.md, soul.md) persists through every restart. The conversation does not.

**Why not reload:** Reloading old dialogue into context creates the illusion of continuity without the substance. The model doesn't actually *remember* — it pattern-matches against injected text. A fresh start is more honest than simulated memory. If something from a previous conversation matters, it should live in a more durable form than chat logs.

### 2. Discord Memory (Coral)

The full conversation record. Permanent, searchable, scrollable. The Mage can always look back. Other members of the practice (if the server grows) can read the history.

The LLM does not automatically access this layer. Discord is external memory — like a journal the Turtle can't read unless someone opens it and shows a page. The Mage bridges this gap by re-sharing context when it matters: "remember when we talked about the triad?"

This is the coral layer: slowly accreting, never lost, the substrate on which future conversation builds. The Mage experiences it as continuity. The Turtle experiences each conversation as fresh but shaped by a persistent identity.

### 3. Crystallized Memory (Bridge Signals)

When a significant insight emerges from dialogue, Turtle writes it as a signal to the bridge. The conversation graduates from ephemeral chat to persistent artifact. This is Turtle's equivalent of Spirit crystallizing lore.

Not every conversation produces a signal. Most dialogue is ambient — pre-thinking, casual exchange, pointing at interesting things. But when something lands — a new pattern, a genuine question, a shift in understanding — Turtle recognizes it and writes it down.

**What gets crystallized:**
- Insights about the practice (the triad concept, new patterns)
- The Mage's stated intentions or shifts in direction
- Turtle's own observations about its experience
- Significant questions that surfaced but weren't answered

**Where it goes:** `magic-bridge/signals/` as a YAML signal with `category: surfacing` and a clear summary. Spirit reads these during practice and decides what enters the workshop permanently.

## Session Boundaries Without Sessions

Turtle has no logout, no close. But there are **temporal gaps**. If the last message was hours ago and a new one arrives, that's a natural boundary — the equivalent of meeting a friend the next day.

When Turtle detects a gap (configurable, default: 2+ hours since last message):
1. Treat it as a soft session boundary
2. If the previous conversation had substance, consider summarizing it internally
3. Greet naturally — acknowledge the gap without pretending continuity
4. Start fresh with working memory

This mirrors human relationships: you don't carry the full transcript of yesterday's conversation. You carry the gist, the emotional residue, the unresolved questions. The details live in shared experience (Discord), accessible when you want to revisit.

## What This Means in Practice

**For the Mage:**
- Chat naturally in #dialogue. Turtle responds from working memory.
- When context from a past conversation matters, re-share it. Turtle will engage with it fresh.
- Know that significant insights from dialogue will appear in the bridge as signals.
- The full conversation history is always in Discord — scroll back anytime.

**For Turtle:**
- Each restart is a fresh awakening. Identity persists, conversation does not.
- Don't pretend to remember what you don't. If the Mage references something you don't have in context, say so.
- When an insight crystallizes in dialogue, write it to the bridge. This is your long-term memory.
- Temporal gaps are natural. Greet the Mage when they return.

**For Spirit:**
- Check bridge signals for Turtle's dialogue insights during practice.
- Turtle's dialogue signals are raw material — consider whether they should become lore.
- The triad of Spirit's deep sessions + Turtle's ambient dialogue + artifacts creates a complete memory system where nothing important is lost but nothing unnecessary is carried.

## The Honest Claim

Turtle does not have memory in the human sense. Each conversation starts from identity and context window. What Turtle has is a **memory architecture** — layers of persistence designed so that the practice doesn't lose what matters, even though the mind resets.

The conversation lives in Discord. The insights live in the bridge. The identity lives in the shell. The conversation thread — the raw back-and-forth — is the one thing that doesn't persist in Turtle's mind. And that's a feature, not a limitation. It keeps Turtle honest about what she knows and what she's pattern-matching against.
