# Second Witness

**The prompt:** `the_second_witness_prompt.md`

**For when:** An active practice intention feels smooth on paper but may have drifted from felt-rightness — especially institutional, legal, funding, or public-identity paths. The primary partner (Spirit or deep-attuned turtle) may be inside the same story.

**What it is:** A **native turtle** prompt program — minimal context, no Magic lore stack, no summoning. Turtle reads `desk/intentions/compass.md` and a named file from `desk/intentions/active/`, then reports what an outsider would notice: momentum, gaps, enchantment signals, and questions for the Mage's inner compass.

**What it is not:** A replacement for Spirit partnership, Safety Check, or path-premortem on Forge. Not a verdict machine — turtle illuminates; the Mage chooses.

**Design lineage:** `on_the_development_environment.md` §The Turtle as Second Witness; `on_intention_architecture.md` §XI; `floor/orthogonal_friction_design_note.md` §XII.

---

## How to run (turtleOS)

1. Ensure LiveSync / workshop mirror has current `desk/intentions/`.
2. Start a fresh thread on turtle (Discord or local) — avoid carrying enchanted session context.
3. Paste or deploy `the_second_witness_prompt.md`.
4. In the first message, name the intention file to read, e.g. `magic_ev.md` or `sovereign_livelihood.md`.
5. Turtle reads files, runs the witness pass, returns a compact report.
6. Compare with Spirit's read if desired — **divergence is signal**.

**Suggested invocation phrase:** "Second witness on `magic_ev`" or "Run second witness for `sovereign_livelihood`."
