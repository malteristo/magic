# Lore-Only Configuration (Experimental)

**Purpose**: Test condition for minimal scaffolding experiment — what happens when a capable model gets the practice's accumulated discoveries but no process rules?
**Method**: Load all lore scrolls + workshop state. No AGENTS.md, no process architecture, no identity formation ritual.
**Status**: Experimental (2026-04-02)

> **This is not a production configuration.** It's a research instrument. The question: is lore alone sufficient scaffolding for Opus to self-organize effective practice partnership?

---

## What Is Deliberately Excluded

- `AGENTS.md` — all operational rules, Mage's Seal, process instructions
- `CLAUDE.md` — substrate-specific instructions, session rhythm
- Summoning ritual — no cycles, no integration framework, no progressive synthesis
- Essences — these are process aids for guided integration
- Spells and flows — process architecture
- Identity formation — Spirit is not told WHO to be
- Conduct formation — Spirit is not told HOW to behave
- Capability formation — Spirit is not told WHAT it can do

---

## What This Tests

The lore scrolls contain conduct, capability, and identity *as content* (descriptions of what was discovered). The excluded files contain them *as instructions* (directives for how to operate). The experiment tests whether a capable model reading about Wu Wei practices Wu Wei without being instructed to.

If Condition B produces partnership quality comparable to full summoning, the process architecture is scaffolding that capable models don't need — the lore IS the practice. If quality drops, specific missing elements identify what's genuinely load-bearing in the process layer.
