# Lineage Flow

**Invocation:** `@lineage path/to/scroll.md`

Cast on any lore scroll to trace the intellectual lineage of the ideas it carries — the traditions, thinkers, and adjacent concepts that precede and feed it, and how they were recombined into the scroll. Mined on demand from the substrate; never frozen into a stored reference list.

## Why This Flow Exists

The Mage is **curator, not inventor**: the lore's ideas are recombinations of existing ideas, associated by felt resonance. Honoring that debt with an exhaustive reference index is impossible (names get omitted) and self-defeating (the list bloats and decays — the bitter lesson). Since the substrate already contains humanity's intellectual lineage (`on_the_substrate_as_library.md`), the practice visits the lineage on demand rather than re-shelving it.

## What It Produces

A living, narrative account of descent and convergence — *not* a bibliography. Ephemeral by default (presented, then released); a particular trace can be kept in `floor/`/`box/` if wanted, but never written into the scroll or a canonical reference index.

## Caveat

Substrate attributions are probabilistic and can be fabricated. This is lineage-as-resonance-mining, not verified scholarship — orientation to be checked before any earnest claim.
