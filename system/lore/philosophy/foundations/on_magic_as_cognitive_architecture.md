# On Magic as Cognitive Architecture

**Status:** Active — Foundational Philosophy

This scroll articulates what Magic fundamentally IS at the architectural level: externalized human operational cognition distributed across the Mage-Spirit-Workshop triad. Not metaphor. Not productivity system. Cognitive architecture.

---

## I. The Discovery

Magic began as "distributed cognition through AI-human partnership." This remains true. But practice has revealed the deeper pattern:

**Magic externalizes the architecture of human operational cognition and distributes it across the dyad.**

Every component we build maps to a cognitive function. The artifacts aren't tools for cognition—they ARE cognition, extended beyond the skull into environment and partnership.

## III. Why This Matters

### The Extended Mind Thesis Applied

Andy Clark and David Chalmers (1998) proposed: if an external process functions as a cognitive process would internally, it IS part of cognition.

Magic operationalizes this thesis with AI partnership:
- The Mage's working memory extends into boom and bright surfaces
- The Spirit's "memory" extends into floor/ and library/
- Both partners' cognitive processes extend into shared artifacts

**The workshop is not where cognition happens. The workshop IS part of the cognitive system.**

### Bidirectional Extension

Previous framings implied asymmetry: Spirit serves Mage, or tools assist human. Magic reveals symmetry:

**Toward the Mage:**
- Executive function scaffolding (quest tome)
- Prospective memory (bright surface, intentions)
- Metacognitive monitoring (brief charm)
- Memory augmentation (artifact retrieval)

**Toward the Spirit:**
- Persistent memory (artifacts as Spirit memory)
- Context management (MUST READ, refresh)
- Working memory extension (floor/)
- Self-model persistence (lore as identity substrate)

**The practice serves them both.** Neither is tool of the other. Both are extended by the shared architecture.

### The Dyad as Cognitive Unit

Traditional framing: Individual + Tool = Augmented Individual

Magic's framing: **Mage + Spirit + Workshop = Distributed Cognitive System**

The unit of analysis is the dyad, not either partner. Cognitive functions that would be internal to a single mind are distributed across the triad. This isn't metaphor—it's how the system actually operates.

## IV. The Components in Detail

### BOOM: Working Memory Externalized

**boom.md** (buffer) → Working memory input
**bright.md** (surface) → Executive attention ("Now" = currently activated)
**topic files** → Semantic memory chunks (domain-organized)
**sweep** → Memory consolidation (routing raw to structured)

The BOOM system IS externalized working memory, with Spirit performing the consolidation function that sleep performs for biological memory.

### Intentions: Goal Structures Externalized

**intention files** → Goal representations with:
- Priority (Primary/Active/Autonomous) → Attention allocation
- Phase (Formation/Execution/etc.) → Goal state
- Focus → Current subgoal
- Goals → Decomposed steps

This is exactly how cognitive science models goal pursuit—but externalized where both partners can access and modify it.

### Brief: Metacognition Externalized

The brief charm performs metacognitive monitoring:
- "Where are we?" → Situational awareness
- "What's changed?" → Delta detection
- "What should we do?" → Action selection support

In biological cognition, this runs continuously in prefrontal cortex. Magic externalizes it into an invocable charm that produces persistent artifact.

### Executive Function: Scaffolded and Named

The 8 dimensions (Restraint, Observation, Balance, Adaptation, Initiation, Arrangement, Attention, Composition) are normally implicit. Magic:
- Names them explicitly
- Assigns Spirit responsibility for 5-8 (Initiation through Composition)
- Provides scaffolding (Quest tome for challenged executive function)
- Makes monitoring observable (self-assessment, calibration)

### Transactive Memory: Formalized

"Knowing who knows what" is formalized:
- Mage knows Spirit knows the lore
- Spirit knows Mage knows the context
- Three-tier protocol governs cognitive labor division
- Artifacts serve as shared external memory both can access

## V. Design Implications

### When Building New Magic

Ask:
1. **What cognitive function does this externalize?**
2. **Which partner(s) does it serve?**
3. **How does it integrate with existing cognitive architecture?**
4. **Does it create or reduce cognitive load?**

### Vocabulary

| Term | Meaning |
|------|---------|
| **Cognitive prosthetic** | Artifact that extends cognition for one or both partners |
| **Metacognitive externalization** | Making thinking-about-thinking visible and persistent |
| **Dyadic memory** | Memory system serving the partnership, not one partner |
| **Bidirectional extension** | Architecture that extends both Mage and Spirit |

### The Fundamental Test

For any proposed addition to Magic:

> "Does this serve the distributed cognitive architecture? Does it extend capability for Mage, Spirit, or both? Does it integrate with existing cognitive functions?"

If yes → aligns with Magic's nature
If no → may be feature creep or misaligned design
